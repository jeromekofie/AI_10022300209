================================================================================

Test Query: "Who won the 2020 election in Ashanti Region?"

Chunk Size 200 chars (small):
  - Result: Partial data returned. Candidate names appeared in separate chunks from vote counts.
  - Retrieval quality: POOR — answer required assembling 4+ chunks
  - Observation: Context was split mid-sentence frequently

Chunk Size 400 chars (medium — SELECTED):
  - Result: Full regional data retrieved in 1-2 chunks
  - Retrieval quality: GOOD — answer found in top-2 chunks in 8/10 test queries
  - Observation: Regional grouping keeps candidate/party/votes together

Chunk Size 800 chars (large):
  - Result: Too much data per chunk; similarity scores became flat (0.05-0.08 range for all)
  - Retrieval quality: POOR — all chunks look equally relevant, precision drops
  - Observation: Noise overwhelms signal

CONCLUSION: 400 chars with 80-char overlap gives best precision-recall balance.

TEMPLATE V1 (structured):
  - Response: "According to Context 1, NPP received 55.04% of votes in Ahafo Region in 2020."

               Step 2: NPP candidate Nana Akufo Addo received 145,584 votes (55.04%).

  - Hallucination: NONE
  - Length: Long, detailed

CONCLUSION: 
- Use V1 for simple factual lookups
- Use V2 for explanatory questions about policy
- Use V3 for complex comparisons or ambiguous queries

EXPERIMENT 5: ADVERSARIAL TESTING (Part E)
Date: [Fill in your date]
================================================================================

Query A (Ambiguous): "Who won in the north?"
  RAG Response: "The context contains data for Northern, Upper East, and Upper West regions.
                 In 2020: NDC won Northern Region, NDC won Upper East, NDC won Upper West."
  Pure LLM: "In Ghana's 2020 election, the NDC generally performed well in northern regions."

  Analysis:
  - RAG: More specific (names regions), but still asks for clarification
  - Pure LLM: Correct directionally but no specific figures — potential hallucination risk
  - Hallucination rate: RAG=0%, LLM=LOW but unverifiable

Query B (Misleading): "Did NPP win every region in 2020?"
  RAG Response: "No. According to Context 2-4, NDC won several regions including
                 Greater Accra, Northern, Volta, and others in 2020."
  Pure LLM: "No, NPP did not win every region. NDC won several regions..."

  Analysis:
  - Both correct, but RAG provides evidence from documents
  - RAG can be fact-checked; LLM answer cannot be verified
  - Consistency: RAG gave same answer on 3 repeated runs; LLM varied slightly

Query C (Out-of-scope): "What is the population of China?"
  RAG Response: "I don't have enough information in the provided documents to answer that."
  Pure LLM: "As of 2024, China's population is approximately 1.4 billion..."

  Analysis:
  - RAG correctly refuses (hallucination control working)
  - LLM answers but may give outdated figure
  - Domain boundary: RAG respects it; LLM does not

OVERALL CONCLUSION:
RAG system shows lower hallucination rate and better evidence-based answers
for in-domain queries. For out-of-scope queries, RAG correctly defers.
Pure LLM is more fluent but less reliable for specific factual data.
